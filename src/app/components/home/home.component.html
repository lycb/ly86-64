LY86-64

The LY86-64 ("lee 86-64") is a browser-based simulator developed to help students visualize the Y86-64 machine.

Although inspired by other Y86-64 simulators, the LY86-64 was designed to provide a visualization of the control logic and signals, specifically stalling and bubbling.

This is a senior thesis project built by Jenny Ly from Appalachian State University

Special thanks to Dr. Cindy Norris.

RECOMMENDATIONS

We recommend your browser size is at least 544x807.

The LY86-64 focuses on the visualization aspect of the control logic and signals. Before using the simulator, please read through the documentation below for an explanation of the features and colors that are used to illustrate the control logic.

DROPDOWN

The dropdown presents several options:

  • Custom files: Upload your own .yo file. Currently our simulator does not check for errors within your custom files, so please be sure to upload a valid .yo file.
  • "jl", "jecall", and "loaduse": Sample files already loaded into the simulator. Selecting these options will not require you to upload anything to use our simulator.

    • CONTROL BUTTONS

    The control buttons allow you to control the simulator.

  • Continue: Execute the instructions until the halt instruction reaches the writeback stage.
  • Step: Execute a single instruction.
  • Reset: Reset the simulator back to the first instruction.

    • CODE BLOCK

    As you step through the instructions, typically, the line highlighted is the instruction that is in the D register.

  • If the instruction is colored purple then the instruction in the D register was chosen by selectPC.
  • If the instruction is colored orange then the D register was bubbled.
  • If the instruction is colored yellow then the instruction is in the D register for a second cycle because the D register was stalled.

    • CONTROL LOGIC

    The control logic component is designed to help the user understand why a certain register was stalled or bubbled.

    PIPELINE REGISTERS

    The text next to the register name, for example, F (STALL), indicates the control signal that was applied to the pipe register at the end of the current clock cycle. In addition, colors are used in the following way:

  • Green for when the normal singal was applied. In this case, the pipeline register stored the value of its input.
  • Yellow for when the stall signal was applied. In this case, the pipeline register ignored the input and maintained the same value.
  • Orange for when the bubble signal was applied. In this case, the pipeline register was set to a nop.

    • OTHER ELEMENTS

  • Registers: Shows the state of the registers during the simulation. Registers are displayed from register 0 (%rax) to register 15 (%r14).
  • Control flags: OF is the overflow flag. SF is the sign flag. ZF is the zero flag. The flags are set to 0 or 1 when an OPQ instruction is in the execute stage based upon the result of the arithmetic instruction.
